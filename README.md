# Introduction

I am using GitLab for the first time and at the beginning it felt very complicated. I had a hard time learning how to use GitLab because there are many different features and tools. However, after practicing and exploring the platform, I started to understand how it works. Even though I understand it better now, I am still learning and improving my skills in using GitLab.
The first thing I learned was how to update my personal information and upload pictures in the About Me section of my GitLab profile. This section allows users to introduce themselves and show some information about their work or interests.
While working on my profile, I found that uploading pictures in GitLab using links from Google Drive is quite easy once you understand the steps.

Firstly, we have to select a photo in Google Drive and then go to the Share option and copy the link of the image.
This link will allow the image to be accessed and displayed on another platform like GitLab.

<img class="profile-photo" src="https://drive.google.com/thumbnail?id=1xmxjfEAhPYSSu4ftZQkUHV5NVaHnX8iE&sz=w400" alt="Profile Photo">

Then we have to copy a specific part of the link.
From the Google Drive sharing link, we only copy the image ID part of the link, which is the section that identifies the specific file in Google Drive. This part will be used to generate the image link that GitLab can display.

<img class="profile-photo" src="https://drive.google.com/thumbnail?id=1a-xbH8y_wbSihZsNmrLFBb62aRgscPc3&sz=w400" alt="Profile Photo">

After that, we go to the GitLab page and find an image link format like this.
In the GitLab Markdown editor, there will already be an image link template. In that link, we need to replace the existing ID with the image ID that we copied from Google Drive. After replacing that part of the link, the image will appear correctly in the GitLab page.
This allows the picture from Google Drive to be displayed in the GitLab documentation or About Me section.
